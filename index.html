This Example shows an interesting use case for the library mostrojs:

As you can see in the code, in file rules.js, you can run any action in a thread apart, simply wrapping action with Engine.Thread(), without change your code

of course the limitation for js web workers are the same, so you cannot wrap an action which, for example, manipulates the dom.

The code for actions remain identically and the rule fired when actions returns are the same.

This simple application show the time elapsed after the first action call, then after the second action call and the last is after coordination rule action.
